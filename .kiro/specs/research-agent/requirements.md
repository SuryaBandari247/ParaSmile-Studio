# Requirements Document

## Introduction

The Research Agent is a Python module that discovers global trending topics across multiple free data sources for automated video production. Rather than focusing solely on software/IT content (an exhausted market), the Research Agent casts a wide net across finance, politics, natural events, AI, science, human interest, and technology. It aggregates trends from Google Trends, Reddit, Yahoo Finance, Wikipedia Current Events, and YouTube Data API, then uses GPT-4o-mini to synthesize raw trends into compelling story pitches. A human-in-the-loop step allows the content producer to select the most promising pitch, which becomes the input to the Script Generator. This module serves as the data foundation for the Faceless Technical Media Engine, ensuring all content is grounded in real-time, data-driven insights rather than generic AI-generated content.

## Glossary

- **Research_Agent**: The Python module responsible for aggregating and analyzing trends from multiple global data sources
- **YouTube_API_Client**: The component that interfaces with YouTube Data API v3
- **Google_Trends_Client**: The component that scrapes Google Trends for real-time trending topics globally (free, no API key required)
- **Reddit_Client**: The component that fetches trending posts from public Reddit subreddits via the public JSON API (no API key required for read-only access)
- **Yahoo_Finance_Client**: The component that fetches market data, stock movers, and sector trends from Yahoo Finance (free, no API key required)
- **Wikipedia_Events_Client**: The component that fetches curated daily global news summaries from Wikipedia Current Events (free, no API key required)
- **Topic_Analyzer**: The component that processes responses from all data sources to identify trending topics across all domains
- **Trend_Score**: A numerical value (0-100) representing the trending strength of a topic based on source count, engagement metrics, and recency
- **Trending_Topic**: A subject matter trending across any domain: finance, politics, natural events, AI, science, human interest, technology, or other global topics
- **API_Rate_Limiter**: The component that manages API quota consumption across all data sources
- **Topic_Cache**: Local storage for previously fetched trending topics with timestamps
- **Macro_Mode**: An operational mode that fetches content from authority channels (business/finance news) to identify macro trends
- **Authority_Channel**: A curated YouTube channel known for business, finance, or technology news (e.g., The Economist, Bloomberg Tech)
- **Cross_Reference_Engine**: A component that identifies topic overlap across all data sources using keyword matching and semantic similarity
- **Finance_Context**: Metadata containing stock tickers and financial entities mentioned in content
- **Story_Pitch**: A compelling video question or angle generated by GPT-4o-mini from raw trend data, containing a title, hook, source trends, context type, and a brief explanation of why the topic is interesting
- **Story_Pitch_Board**: A ranked list of 10-15 Story_Pitches presented to the content producer for human-in-the-loop selection
- **Pitch_Generator**: The component that uses OpenAI GPT-4o-mini to synthesize raw trends into Story_Pitches
- **Context_Type**: A classification of a Story_Pitch as either "recent_event" (real-time data available) or "historic_topic" (historical context will be included)
- **Source_Trend**: A reference to the raw trend data from one or more data sources that informed a Story_Pitch

## Requirements

### Requirement 1: YouTube API Integration

**User Story:** As a content producer, I want to connect to YouTube Data API, so that I can access real-time video data for trend analysis.

#### Acceptance Criteria

1. THE YouTube_API_Client SHALL authenticate using a valid API key from environment variables
2. WHEN the API key is missing or invalid, THE YouTube_API_Client SHALL raise an AuthenticationError with a descriptive message
3. THE YouTube_API_Client SHALL use YouTube Data API v3 endpoints
4. WHEN API quota is exceeded, THE YouTube_API_Client SHALL return a QuotaExceededError with the reset timestamp

### Requirement 2: Trending Topic Discovery

**User Story:** As a content producer, I want to discover trending topics across all domains, so that I can create relevant high-RPM videos on globally interesting subjects.

#### Acceptance Criteria

1. WHEN querying for trends, THE Topic_Analyzer SHALL search videos published within the last 7 days
2. THE Topic_Analyzer SHALL accept topics from any domain including finance, politics, natural events, AI, science, human interest, and technology
3. THE Topic_Analyzer SHALL retrieve at least 50 videos per query to ensure statistical significance
4. WHEN a video matches multiple categories, THE Topic_Analyzer SHALL assign it to the category with highest relevance score
5. THE Research_Agent SHALL return topics ranked by Trend_Score in descending order

### Requirement 3: Trend Scoring Algorithm with Macro Mode

**User Story:** As a content producer, I want topics scored by trending strength with macro-economic context, so that I can prioritize high-potential content that aligns with broader global trends.

#### Acceptance Criteria

1. THE Topic_Analyzer SHALL calculate Trend_Score using source count (number of data sources reporting the topic), engagement metrics, and recency
2. WHEN calculating Trend_Score, THE Topic_Analyzer SHALL apply a recency weight where topics surfacing within 24 hours receive a 2x multiplier
3. THE Topic_Analyzer SHALL normalize Trend_Score to a range of 0-100
4. WHEN two topics have identical Trend_Scores, THE Topic_Analyzer SHALL rank by total engagement count
5. THE Research_Agent SHALL exclude topics with Trend_Score below 30
6. WHEN macro_mode is enabled, THE Research_Agent SHALL fetch the latest 5 videos from each configured Authority_Channel
7. THE Research_Agent SHALL maintain a list of Authority_Channel IDs including: The Economist, Bloomberg Technology, CNBC Television, Financial Times, Wall Street Journal, TechCrunch, and The Verge
8. THE Cross_Reference_Engine SHALL identify topic overlap across all data sources using keyword matching and semantic similarity
9. WHEN a topic appears in 3 or more data sources, THE Topic_Analyzer SHALL apply a 2x Trend_Score bonus (multiplicative with other bonuses)
10. THE Research_Agent SHALL extract stock tickers (format: $SYMBOL) from content titles, descriptions, and tags
11. THE Research_Agent SHALL include a finance_context key in the output JSON containing: stock_tickers (list), mentioned_companies (list), and macro_relevance_score (0-1)
12. WHEN macro_mode is disabled, THE Research_Agent SHALL operate in standard mode without authority channel fetching or cross-referencing

### Requirement 4: API Rate Limiting and Quota Management

**User Story:** As a system operator, I want API quota managed efficiently, so that I can avoid service interruptions.

#### Acceptance Criteria

1. THE API_Rate_Limiter SHALL track daily quota consumption against the YouTube API limit of 10,000 units
2. WHEN quota consumption exceeds 80% of daily limit, THE API_Rate_Limiter SHALL log a warning with remaining quota
3. WHEN quota consumption reaches 95% of daily limit, THE API_Rate_Limiter SHALL prevent new API calls until quota reset
4. THE API_Rate_Limiter SHALL calculate the cost of each API operation before execution
5. WHEN the API returns a 403 quota error, THE API_Rate_Limiter SHALL cache the error and prevent retries for 1 hour

### Requirement 5: Topic Caching and Freshness

**User Story:** As a content producer, I want recent trend data cached, so that I can reduce API calls and improve response time.

#### Acceptance Criteria

1. THE Topic_Cache SHALL store trending topics with a timestamp of when they were fetched
2. WHEN cached data exists and is less than 6 hours old, THE Research_Agent SHALL return cached results without making API calls
3. WHEN cached data is older than 6 hours, THE Research_Agent SHALL fetch fresh data from YouTube API
4. THE Topic_Cache SHALL persist data to disk in JSON format
5. WHEN the cache file is corrupted or unreadable, THE Research_Agent SHALL delete the cache and fetch fresh data

### Requirement 6: Topic Classification

**User Story:** As a content producer, I want topics classified by domain category, so that I can target specific audience segments and diversify content.

#### Acceptance Criteria

1. THE Topic_Analyzer SHALL classify topics into categories: Finance, Politics, Natural Events, AI and Technology, Science, Human Interest, Geopolitics, Economics, and Uncategorized
2. WHEN analyzing content metadata, THE Topic_Analyzer SHALL use titles, descriptions, and tags for classification
3. THE Topic_Analyzer SHALL assign a confidence score (0-1) to each classification
4. WHEN classification confidence is below 0.6, THE Topic_Analyzer SHALL mark the topic as "Uncategorized"
5. THE Research_Agent SHALL return topics grouped by category

### Requirement 7: Search Query Configuration

**User Story:** As a content producer, I want to configure search parameters, so that I can customize trend discovery for different niches.

#### Acceptance Criteria

1. THE Research_Agent SHALL accept a list of seed keywords for topic discovery
2. WHERE custom date ranges are specified, THE Research_Agent SHALL search videos within the provided date range
3. WHERE minimum view count is specified, THE Topic_Analyzer SHALL filter out videos below the threshold
4. THE Research_Agent SHALL support configuration via a YAML or JSON configuration file
5. WHEN no configuration file is provided, THE Research_Agent SHALL use default parameters: 7-day window, 1000 minimum views, and predefined technical keywords

### Requirement 8: Error Handling and Resilience

**User Story:** As a system operator, I want robust error handling, so that temporary failures do not crash the system.

#### Acceptance Criteria

1. WHEN YouTube API returns a 5xx server error, THE YouTube_API_Client SHALL retry the request up to 3 times with exponential backoff
2. WHEN network connectivity is lost, THE YouTube_API_Client SHALL raise a NetworkError after 30 seconds timeout
3. IF API response parsing fails, THEN THE Research_Agent SHALL log the raw response and raise a ParseError
4. WHEN any component raises an exception, THE Research_Agent SHALL log the full stack trace with context
5. THE Research_Agent SHALL continue processing remaining topics when a single topic analysis fails

### Requirement 9: Output Format and Data Structure

**User Story:** As a downstream consumer, I want structured topic data with source attribution and financial context, so that I can integrate it with the story pitch pipeline and understand data provenance.

#### Acceptance Criteria

1. THE Research_Agent SHALL return topic data as a Python dictionary with keys: topic_name, category, trend_score, source_count, sources, top_videos, fetched_at, and finance_context
2. WHEN exporting to JSON, THE Research_Agent SHALL format timestamps in ISO 8601 format
3. THE Research_Agent SHALL include up to 5 top_videos per topic with metadata: video_id, title, channel, view_count, and published_at
4. THE Research_Agent SHALL validate output schema before returning results
5. WHEN validation fails, THE Research_Agent SHALL raise a SchemaValidationError with details of the violation
6. THE finance_context dictionary SHALL contain keys: stock_tickers (list of strings), mentioned_companies (list of strings), macro_relevance_score (float 0-1), and authority_channel_match (boolean)
7. WHEN no financial entities are detected, THE finance_context SHALL contain empty lists and a macro_relevance_score of 0.0
8. THE sources list SHALL contain entries with keys: source_name (string), source_url (string), and fetched_at (ISO 8601 timestamp)

### Requirement 10: Logging and Observability

**User Story:** As a system operator, I want detailed logging, so that I can debug issues and monitor system health.

#### Acceptance Criteria

1. THE Research_Agent SHALL log all API requests with timestamp, endpoint, and quota cost
2. THE Research_Agent SHALL log trend analysis results including topic count and average Trend_Score
3. WHEN errors occur, THE Research_Agent SHALL log at ERROR level with full context
4. THE Research_Agent SHALL support configurable log levels: DEBUG, INFO, WARNING, ERROR
5. WHERE structured logging is enabled, THE Research_Agent SHALL output logs in JSON format with consistent field names

### Requirement 11: Google Trends Integration

**User Story:** As a content producer, I want to discover real-time trending topics from Google Trends, so that I can identify globally popular subjects as they emerge.

#### Acceptance Criteria

1. THE Google_Trends_Client SHALL fetch daily trending searches and real-time trending topics without requiring an API key
2. THE Google_Trends_Client SHALL support geographic filtering by country code (default: global)
3. WHEN fetching trends, THE Google_Trends_Client SHALL retrieve trending topics from the last 24 hours by default
4. THE Google_Trends_Client SHALL return each trend with fields: topic_name, approximate_search_volume, related_queries (list of strings), and source_url
5. WHEN Google Trends is unreachable, THE Google_Trends_Client SHALL log a warning and continue with other data sources
6. THE Google_Trends_Client SHALL normalize topic names to lowercase for cross-source deduplication

### Requirement 12: Reddit Public API Integration

**User Story:** As a content producer, I want to discover trending discussions from Reddit, so that I can identify topics generating public interest across diverse domains.

#### Acceptance Criteria

1. THE Reddit_Client SHALL fetch top and hot posts from public subreddits using the Reddit JSON API without requiring an API key
2. THE Reddit_Client SHALL monitor subreddits across multiple domains including: worldnews, economics, technology, science, futurology, geopolitics, finance, dataisbeautiful, and explainlikeimfive
3. THE Reddit_Client SHALL retrieve the top 25 posts per subreddit sorted by hot ranking
4. THE Reddit_Client SHALL extract from each post: title, score, comment_count, subreddit, permalink, and created_utc
5. WHEN a subreddit is private or unavailable, THE Reddit_Client SHALL skip the subreddit, log a warning, and continue with remaining subreddits
6. THE Reddit_Client SHALL respect Reddit rate limits by enforcing a minimum interval of 2 seconds between requests
7. THE Reddit_Client SHALL set a descriptive User-Agent header to comply with Reddit API guidelines

### Requirement 13: Yahoo Finance Integration

**User Story:** As a content producer, I want to discover trending market data and stock movers, so that I can identify finance-related stories with real data backing.

#### Acceptance Criteria

1. THE Yahoo_Finance_Client SHALL fetch market summary data, top gainers, top losers, and most active stocks without requiring an API key
2. THE Yahoo_Finance_Client SHALL retrieve sector performance data to identify trending economic sectors
3. THE Yahoo_Finance_Client SHALL return each market mover with fields: symbol, name, price, change_percent, volume, and sector
4. WHEN a stock moves more than 5% in a single day, THE Yahoo_Finance_Client SHALL flag the stock as a potential story trigger
5. WHEN Yahoo Finance is unreachable, THE Yahoo_Finance_Client SHALL log a warning and continue with other data sources
6. THE Yahoo_Finance_Client SHALL enrich the Research_Agent finance_context with real-time price data for any stock tickers detected across other sources

### Requirement 14: Wikipedia Current Events Integration

**User Story:** As a content producer, I want to access curated daily global news summaries from Wikipedia, so that I can identify verified notable events for video topics.

#### Acceptance Criteria

1. THE Wikipedia_Events_Client SHALL fetch the current day's events from the Wikipedia Current Events portal without requiring an API key
2. THE Wikipedia_Events_Client SHALL parse event entries into structured records with fields: headline, category, date, related_links (list of strings), and summary
3. THE Wikipedia_Events_Client SHALL classify events into categories: Armed Conflicts, Disasters, International Relations, Law and Crime, Politics, Science and Technology, and Sports
4. WHEN the current day's events page is unavailable, THE Wikipedia_Events_Client SHALL fall back to the previous day's events
5. THE Wikipedia_Events_Client SHALL extract named entities (people, places, organizations) from event summaries for cross-source matching

### Requirement 15: Cross-Source Trend Aggregation

**User Story:** As a content producer, I want trends from all sources merged and deduplicated, so that I can see a unified view of globally trending topics.

#### Acceptance Criteria

1. THE Cross_Reference_Engine SHALL merge topics from Google Trends, Reddit, Yahoo Finance, Wikipedia Current Events, and YouTube into a unified topic list
2. THE Cross_Reference_Engine SHALL deduplicate topics using keyword matching and semantic similarity with a similarity threshold of 0.75
3. WHEN the same topic appears in multiple sources, THE Cross_Reference_Engine SHALL merge the entries into a single Trending_Topic with a combined source list
4. THE Cross_Reference_Engine SHALL calculate a cross-source Trend_Score that weights topics appearing in more sources higher than single-source topics
5. WHEN a topic appears in 3 or more sources, THE Cross_Reference_Engine SHALL mark the topic as "high_confidence"
6. THE Cross_Reference_Engine SHALL preserve the original source metadata for each merged topic in the sources list
7. THE Research_Agent SHALL return the unified topic list sorted by cross-source Trend_Score in descending order

### Requirement 16: Story Pitch Generation

**User Story:** As a content producer, I want raw trends synthesized into compelling video story pitches by GPT-4o-mini, so that I can quickly evaluate which topics make the best videos.

#### Acceptance Criteria

1. THE Pitch_Generator SHALL accept the unified Trending_Topic list from the Cross_Reference_Engine as input
2. THE Pitch_Generator SHALL call OpenAI GPT-4o-mini to generate a Story_Pitch_Board containing 10 to 15 Story_Pitches
3. EACH Story_Pitch SHALL contain: title (a compelling question or angle), hook (one-sentence summary of why this is interesting), source_trends (list of Source_Trend references), context_type (either "recent_event" or "historic_topic"), and category
4. WHEN the context_type is "recent_event", THE Pitch_Generator SHALL include a note indicating real-time data is available from the source trends
5. WHEN the context_type is "historic_topic", THE Pitch_Generator SHALL include a note indicating historical context will be included in the script
6. THE Pitch_Generator SHALL instruct GPT-4o-mini to generate pitches as specific, curiosity-driven questions or provocative angles rather than generic topic summaries
7. THE Pitch_Generator SHALL instruct GPT-4o-mini to avoid clickbait language, hyperbole, and generic phrasing
8. THE Pitch_Generator SHALL read the OpenAI API key from the environment variable OPENAI_API_KEY
9. WHEN the OPENAI_API_KEY is missing or invalid, THE Pitch_Generator SHALL raise an AuthenticationError with a descriptive message
10. THE Pitch_Generator SHALL rank Story_Pitches by estimated audience interest based on source Trend_Scores and source count

### Requirement 17: Human-in-the-Loop Topic Selection

**User Story:** As a content producer, I want to browse and select from generated story pitches, so that I maintain creative control over which topics become videos.

#### Acceptance Criteria

1. THE Research_Agent SHALL present the Story_Pitch_Board to the content producer as a numbered list with title, hook, context_type, and category for each pitch
2. THE Research_Agent SHALL accept a pitch selection by index number from the content producer
3. WHEN a valid pitch index is selected, THE Research_Agent SHALL return the selected Story_Pitch as a structured Python dataclass with all fields populated
4. WHEN an invalid pitch index is provided, THE Research_Agent SHALL display an error message and re-prompt for selection
5. THE Research_Agent SHALL support a "regenerate" command that triggers a new Story_Pitch_Board from the same raw trends
6. THE Research_Agent SHALL support a "details" command followed by a pitch index that displays the full source trend data for that pitch
7. THE selected Story_Pitch SHALL serve as the input to the Script Generator module
