# Requirements Document

## Introduction

The Pipeline UI is a lightweight local web application that provides a human-in-the-loop interface for the Faceless Technical Media Engine. It exposes the full content production pipeline — from trend research through script conversion to VideoScript review — as a step-by-step guided workflow. The UI allows the content producer to trigger the Research Agent, browse and select story pitches, paste raw scripts, upload supporting documents (PDF, XLSX, CSV, TXT, MD, DOCX), convert scripts to structured VideoScript JSON via GPT-4o-mini, and review the final output before handing it to the Asset Orchestrator. This is a local development tool, not a production SaaS, so simplicity and fast iteration are prioritized over scalability.

## Glossary

- **Pipeline_UI**: The web application that orchestrates the end-to-end content production workflow through a step-by-step interface
- **Pipeline_State**: The current position in the workflow, one of: search, select_topic, script_input, convert, review
- **Search_Panel**: The UI component that triggers the Research Agent and displays the Story_Pitch_Board
- **Topic_Card**: A UI element displaying a single Story_Pitch with title, hook, context_type, category, and source trend count
- **Script_Input_Panel**: The UI component where the user pastes raw script text and uploads supporting documents
- **Document_Parser**: The component that extracts plain text content from uploaded files (PDF, XLSX, CSV, TXT, MD, DOCX)
- **Conversion_Panel**: The UI component that triggers the Script Converter with the raw script and extracted document context
- **Review_Panel**: The UI component that displays the structured VideoScript for review before rendering
- **Research_Agent**: The existing Python module that discovers trending topics and generates Story_Pitches (from research_agent/)
- **Script_Converter**: The existing Python module that converts raw text to VideoScript JSON via GPT-4o-mini (from script_generator/)
- **Video_Script**: The structured JSON output containing ordered Scene_Blocks consumed by the Asset Orchestrator
- **Scene_Block**: A single scene unit containing narration_text and a Visual_Instruction
- **Story_Pitch**: A compelling video question or angle generated by GPT-4o-mini from raw trend data
- **Story_Pitch_Board**: A ranked list of 10-15 Story_Pitches presented for human selection
- **Uploaded_Document**: A file uploaded by the user containing supplementary context (research reports, statistical data)
- **Extracted_Text**: Plain text content parsed from an Uploaded_Document and appended to the raw script as LLM context

## Requirements

### Requirement 1: Web Application Framework

**User Story:** As a content producer, I want a lightweight local web UI, so that I can interact with the pipeline through a browser without complex setup.

#### Acceptance Criteria

1. THE Pipeline_UI SHALL be implemented as a Streamlit web application
2. THE Pipeline_UI SHALL start with a single command: `streamlit run pipeline_ui/app.py`
3. THE Pipeline_UI SHALL run locally on the default Streamlit port (8501)
4. THE Pipeline_UI SHALL load configuration from environment variables (OPENAI_API_KEY, YOUTUBE_API_KEY) using python-dotenv
5. IF a required environment variable is missing, THEN THE Pipeline_UI SHALL display an error message identifying the missing variable and prevent pipeline execution

### Requirement 2: Step-by-Step Pipeline Navigation

**User Story:** As a content producer, I want a guided step-by-step workflow, so that I stay in control of each stage of the content production process.

#### Acceptance Criteria

1. THE Pipeline_UI SHALL present the pipeline as a sequential workflow with five steps: Search, Select Topic, Script Input, Convert, Review
2. THE Pipeline_UI SHALL display a progress indicator showing the current step and completed steps
3. THE Pipeline_UI SHALL allow the user to navigate back to any previously completed step
4. WHEN the user navigates back to a previous step, THE Pipeline_UI SHALL preserve the data from that step and all prior steps
5. THE Pipeline_UI SHALL prevent the user from advancing to the next step until the current step is complete
6. THE Pipeline_UI SHALL persist Pipeline_State in Streamlit session state so that browser refreshes do not lose progress within a session

### Requirement 3: Research Agent Integration (Search Step)

**User Story:** As a content producer, I want to trigger trend research from the UI, so that I can discover trending topics without using the command line.

#### Acceptance Criteria

1. WHEN the user clicks the "Search for Trends" button, THE Search_Panel SHALL invoke the Research_Agent to fetch trending topics from all configured data sources
2. WHILE the Research_Agent is executing, THE Search_Panel SHALL display a loading indicator with a status message
3. WHEN the Research_Agent completes, THE Search_Panel SHALL display the Story_Pitch_Board as a list of Topic_Cards
4. EACH Topic_Card SHALL display: title, hook, context_type, category, and the number of source trends
5. IF the Research_Agent raises an error, THEN THE Search_Panel SHALL display the error message and allow the user to retry
6. THE Search_Panel SHALL allow the user to click "Regenerate Pitches" to request a new Story_Pitch_Board from the same raw trends

### Requirement 4: Topic Selection (Select Topic Step)

**User Story:** As a content producer, I want to browse and select a story pitch from the search results, so that I can choose the most promising topic for my next video.

#### Acceptance Criteria

1. THE Pipeline_UI SHALL display the Story_Pitch_Board from the Search step as selectable Topic_Cards
2. WHEN the user clicks on a Topic_Card, THE Pipeline_UI SHALL highlight the selected card and display the full source trend details for that pitch
3. WHEN the user confirms a topic selection, THE Pipeline_UI SHALL store the selected Story_Pitch in Pipeline_State and advance to the Script Input step
4. THE Pipeline_UI SHALL allow the user to change the selection before confirming

### Requirement 5: Raw Script Input (Script Input Step)

**User Story:** As a content producer, I want to paste my externally written script into the UI, so that I can feed it into the conversion pipeline.

#### Acceptance Criteria

1. THE Script_Input_Panel SHALL provide a multi-line text area for pasting raw script text
2. THE Script_Input_Panel SHALL display the selected Story_Pitch title and hook as context above the text area
3. WHEN the text area is empty, THE Pipeline_UI SHALL prevent advancement to the Convert step
4. THE Script_Input_Panel SHALL display a character count for the entered script text
5. THE Pipeline_UI SHALL store the raw script text in Pipeline_State

### Requirement 6: Document Upload and Parsing

**User Story:** As a content producer, I want to upload supporting documents (research reports, data files), so that the LLM has richer context when converting my script to VideoScript format.

#### Acceptance Criteria

1. THE Script_Input_Panel SHALL provide a file upload widget that accepts multiple files
2. THE Document_Parser SHALL support the following file formats: PDF, XLSX, CSV, TXT, MD, DOCX
3. WHEN a file is uploaded, THE Document_Parser SHALL extract plain text content from the file
4. WHEN a PDF file is uploaded, THE Document_Parser SHALL extract text from all pages of the PDF
5. WHEN an XLSX file is uploaded, THE Document_Parser SHALL extract cell values from all sheets, converting each row to a comma-separated text line
6. WHEN a CSV file is uploaded, THE Document_Parser SHALL read the file content as plain text
7. WHEN a TXT or MD file is uploaded, THE Document_Parser SHALL read the file content as plain text with UTF-8 encoding
8. WHEN a DOCX file is uploaded, THE Document_Parser SHALL extract paragraph text from the document
9. IF a file format is not in the supported list, THEN THE Document_Parser SHALL display an error message identifying the unsupported format and skip the file
10. IF document text extraction fails, THEN THE Document_Parser SHALL display a warning message with the filename and error, and continue processing remaining files
11. THE Script_Input_Panel SHALL display the list of successfully parsed documents with their extracted text length in characters
12. THE Pipeline_UI SHALL store all Extracted_Text in Pipeline_State, concatenated with clear document boundary markers

### Requirement 7: Script Conversion (Convert Step)

**User Story:** As a content producer, I want the raw script and uploaded document context converted into a structured VideoScript, so that the Asset Orchestrator can render it.

#### Acceptance Criteria

1. WHEN the user clicks "Convert to VideoScript", THE Conversion_Panel SHALL concatenate the raw script text with all Extracted_Text, separated by a document context header
2. THE Conversion_Panel SHALL invoke the Script_Converter with the concatenated text as input
3. WHILE the Script_Converter is executing, THE Conversion_Panel SHALL display a loading indicator with a status message
4. WHEN the Script_Converter completes, THE Conversion_Panel SHALL store the resulting Video_Script in Pipeline_State and advance to the Review step
5. IF the Script_Converter raises a ParseError or ValidationError, THEN THE Conversion_Panel SHALL display the error details and allow the user to edit the raw script and retry
6. THE Conversion_Panel SHALL display the LLM model used and token counts after conversion completes

### Requirement 8: VideoScript Review (Review Step)

**User Story:** As a content producer, I want to preview the structured VideoScript before rendering, so that I can verify the output is correct.

#### Acceptance Criteria

1. THE Review_Panel SHALL display the Video_Script title, total word count, and generation timestamp
2. THE Review_Panel SHALL display each Scene_Block with its scene number, narration text, and visual instruction details
3. FOR EACH Scene_Block, THE Review_Panel SHALL display the Visual_Instruction type, title, and a formatted view of the data field
4. THE Review_Panel SHALL provide a button to download the Video_Script as a JSON file
5. THE Review_Panel SHALL provide a "Back to Script" button that returns to the Script Input step with the current raw script preserved
6. THE Review_Panel SHALL display the raw JSON output in a collapsible section for inspection

### Requirement 9: Error Handling and User Feedback

**User Story:** As a content producer, I want clear error messages and feedback, so that I can understand and recover from issues in the pipeline.

#### Acceptance Criteria

1. WHEN any pipeline step encounters an error, THE Pipeline_UI SHALL display the error in a visible error banner with a descriptive message
2. THE Pipeline_UI SHALL distinguish between recoverable errors (user can retry) and configuration errors (user must fix environment)
3. WHEN a network or API error occurs, THE Pipeline_UI SHALL suggest checking API keys and network connectivity
4. THE Pipeline_UI SHALL log all errors to the Streamlit logger with full context including the pipeline step and error type

### Requirement 10: Session State Management

**User Story:** As a content producer, I want my progress preserved during a session, so that I do not lose work when navigating between steps.

#### Acceptance Criteria

1. THE Pipeline_UI SHALL store all pipeline data (search results, selected topic, raw script, uploaded documents, converted VideoScript) in Streamlit session state
2. WHEN the user navigates between steps, THE Pipeline_UI SHALL restore the data for each step from session state
3. THE Pipeline_UI SHALL provide a "Start New Pipeline" button that clears all session state and returns to the Search step
4. WHEN the user starts a new pipeline, THE Pipeline_UI SHALL prompt for confirmation before clearing session state
